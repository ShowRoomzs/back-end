비즈니스 규칙
연결됨에서만 계약 시작·소통 작성 가능.
수락 단계 필수 (인플루언서 1-클릭 수락).
진행중 공구가 있으면 해제 제한 — 공구 종료 후 해제 가능.
같은 쌍의 재연결 허용 — 스레드는 쌍 기준이라 이어짐.
같은 쌍에 동시에 유효한 연결은 1개 (연결요청/연결됨 상태 중복 생성 방지).

비즈니스 규칙
스레드 상태 = 연결 상태 파생 — 연결됨=열림 / 해제=휴면(열람만, 작성 불가) / 재연결=다시 열림.
읽음 처리 = 스레드 단위 — 참여자별 "마지막 읽은 시각" 저장 → 그 이후 메시지 수 = 안읽은 수(뱃지). 메시지 개별 읽음표시는 두지 않음.
미디어 첨부 = 이미지·영상, 1글당 최대 20개.
운영자는 열람만 (작성 불가).
새 메시지 → 상대에게 인앱 알림(#5).

비즈니스 규칙
계약 상품 항목이 공구가·리워드율의 원천 — 상품에 박지 않고 여기에 저장(쇼룸/계약마다 다를 수 있으므로).
리워드율은 상품 항목 단위 — 상품마다 다른 리워드율 가능. 정산은 상품 항목별 리워드율로 계산.
공구 기간은 계약 단위 — 계약의 시작·종료일을 공구가 상속.
서명 순서 = 인플루언서 → 브랜드, 부분서명은 서명 여부 플래그로 추적.
체결완료가 공구 생성 게이트 (계약 1:1 공구). 종료 상태(거절·만료·취소)는 불가역.
증거는 A안 솔루션 참조만 보관.

공구가·대상 상품·기간은 계약(상품 항목) 참조 — 공구는 계약ID만 갖고 값을 복사하지 않음. 계약은 체결 후 불변이라 정합.
계약 체결완료 시 생성(준비중).

진행중 = 준비완료 AND 시작일 도래(자동).
공구 종료 → 게시물 자동 종료·장바구니 항목 자동 제거 트리거.

진열 2단 — 상품 단위(미진열=전체 숨김) + 조합 단위(개별 조합 숨김). 진행중 공구 상품의 상품 미진열은 운영자만.
진행중 공구 상품은 정가·핵심 정보 수정 잠금(재고 보충만 허용, 수정은 운영자 경유).
단일 상품(옵션 없음)도 기본 조합 1개로 통일 — 재고·구매 로직 단일화.
검수 없음(등록 즉시 진열).
목업 반영 메모 (화면/개발)
제거: 옵션가 입력(14·21), 대표 옵션 여부(24) — 가격이 조합 무관하므로 불필요.
유지: 옵션 그룹/항목 추가·삭제, 조합 자동 생성(20), 조합별 재고(22)·진열 토글(23).

비즈니스 규칙
담기는 상품 = 공구(계약)의 전체 상품 자동 — 인플루언서가 일부만 빼거나 고르지 않음(계약과 일치).
대가관계 표시 자동 삽입 — 시스템이 표시 배지/문구를 자동 삽입(속성으로 관리 불필요 — 렌더링 규칙), 인플루언서 제거·숨김 불가.
예약·노출중 상태의 본문 수정 잠금 — 수정은 운영자 경유(경량 검수 실효성 보장). 일반 게시물은 자유.
공구당 1개 필수, 본문은 글만.
상세페이지 = 이 게시물(리스팅) 단위 — 상품 상세·공구가는 "게시물 → 공구 → 계약 상품 항목"으로 조회. 소비자는 담긴 상품 중 원하는 것만 개별 구매(묶음 진열 ≠ 강제 구매).
운영자만 숨김. 공구 종료 시 자동 종료. 예약 = 공구 시작일 자동 노출.

배송 — 하위주문(브랜드) 단위. 브랜드 "준비 시작" → 상품준비중, 송장 등록 → 배송중.
구매확정 = 배송완료 + 7일 자동(하위주문 단위). 정산 대상.
단순 취소(결제완료·준비 전) — 소비자 취소 시 PG 자동 취소(운영자 불필요), 취소·반품 엔터티에 기록.
반품(배송완료~구매확정 전) — 요청 → 운영자 승인 → 택배사 회수(문 앞 수거) → 입고(브랜드) → 운영자 PG 취소.
공구 취소·운영자 사유 환불은 단계 무관.
부분 취소 = SKU 항목 + 수량 단위. 다른 옵션은 별개 항목이라 각각 독립.
정산 집계 = 하위주문(공구)별 구매확정 항목 − 취소·반품.
환불 배송지 = 브랜드 반품 수취 주소. 반송비 부담 = 정책(약관).
[확장 가능]
교환(반품+재발송), 반품 승인/거절 이력 상세.
